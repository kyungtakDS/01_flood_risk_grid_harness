# data/standard_grid/ — 이미 만들어진 표준격자망 제공 공간

표준격자망이나 이미 100m 격자로 제작된 자료를 **직접 제공**할 때 넣는 공간. 하네스가 새로 만들지 않고 이것을 재사용한다(단, 계약 검증 후).

## grid_master/
이미 만든 **마스터 격자**(100m, EPSG:5179)를 넣는다. 대권역별로 여러 개 가능.
- 넣으면 geo-data-engineer가 마스터를 새로 생성하지 않고 이 격자를 기준으로 삼는다.
- 없으면 하네스가 수자원단위 대권역별로 마스터를 생성한다.

## pre_gridded/
이미 100m 격자로 제작된 자료(예: 격자화된 침수위험지도·NDMS, 격자 인구)를 넣는다.
- **무검증 사용 금지.** `src/standardize/verify_grid.py`(→ `assert_grid_contract`)로 마스터 정합을 검증한 뒤 사용한다.
- 통과 → 재표준화 없이 그대로 사용(불필요한 리샘플로 값 훼손 방지).
- 위반(원점 밀림·CRS 상이·NoData 규약 차이) → 마스터에 재정합.

**담당:** geo-data-engineer + validation-inspector.

> "이미 100m 격자니까 맞겠지"가 가장 흔한 함정이다. 남이 만든 격자는 원점·CRS 변형·NoData가 다를 수 있어 셀 join을 조용히 어긋나게 한다. 계약 검증은 값싸고 오염 전파는 비싸다.
