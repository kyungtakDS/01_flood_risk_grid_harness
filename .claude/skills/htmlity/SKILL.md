# htmlity — 진행 내용을 읽기 좋은 HTML 문서로 정리·저장

작업 진행 내용·세션 로그·결과·결정사항을 **한 개의 자체완결 HTML 파일**로 구조화해 저장한다. 브라우저로 바로 열리고, 나중에 다시 봐도 "무엇을·왜·어떻게 했는지"가 한눈에 읽힌다.

**왜 이렇게 하는가:** 진행 내용은 대화·터미널에 흩어지면 금방 사라진다. 이를 (1) 시간 순 흐름, (2) 산출물 목록, (3) 결정 근거로 구조화해 남기면, 나중에 이어서 작업하거나 인수인계할 때 재구성 비용이 0에 가깝다. 그래서 htmlity는 "예쁜 페이지"가 아니라 **다시 읽을 수 있는 기록**을 목표로 한다.

## 워크플로우

```
1. 내용 수집  → 검증: 정리할 진행 내용(단계·산출물·결정)을 확보했는가
2. 구조화     → 검증: 아래 권장 섹션으로 배치되는가
3. 렌더링     → 검증: assets/report-template.html 스타일로 자체완결 HTML 조립
4. 저장       → 검증: docs/ 하위(또는 지정 경로)에 저장, 기존 파일이면 갱신
5. 자기검증   → 검증: 단일 파일로 열림·앵커/TOC 동작·외부 의존 없음
```

### 1. 내용 수집
정리 대상을 모은다: 이번 세션(또는 지정 범위)에 **한 일, 만든 파일, 내린 결정과 그 이유, 검증 결과, 남은 일**. 대화·터미널 출력·git 로그·산출 파일에서 사실만 뽑는다. 지어내지 않는다 — 근거가 없으면 "미확정"으로 둔다.

### 2. 구조화 (권장 섹션)
| 섹션 | 담는 것 |
|------|--------|
| 개요 | 무엇을 요청받았고 무엇을 했는지 3~5줄 요약 + 상태 |
| 진행 타임라인 | 시간/순서대로 단계별로: 무엇을 했고 결과는 무엇 |
| 산출물 | 만들거나 바꾼 파일 표(경로 · 상태 · 설명) |
| 결정사항 | 왜 그렇게 했는가(대안 대비 선택 근거). 콜아웃으로 강조 |
| 검증 | 어떻게 확인했는가(테스트·렌더·수치 대조 결과) |
| 다음 단계 | 남은 일·후속 제안 |
| 참고 | 커밋 해시·URL·근거 파일 등 링크 |

없는 섹션은 뺀다. 억지로 채우지 않는다.

### 3. 렌더링
`assets/report-template.html`을 복제해 내용을 채운다. 템플릿은:
- 밝은 배경 · 읽기 좋은 본문 컬럼(max ~760px) · 좌측 **스티키 TOC**(섹션 앵커)
- **타임라인** 컴포넌트, **산출물 표**, **콜아웃**(결정/주의), **상태 pill**, 모노 경로
- **외부 의존 0** — 순수 HTML/CSS(오프라인에서도 열림). 차트가 꼭 필요할 때만 Chart.js CDN 1개 허용
- 밝은 배경 고정(다크 분기 없음), 반응형(모바일에서 TOC 상단 접힘)
스타일 토큰·컴포넌트 마크업은 템플릿 안의 `<!-- htmlity: ... -->` 주석을 따른다.

### 4. 저장
- 기본 경로: `docs/{주제}-{YYYY-MM-DD}.html` (예: `docs/progress-2026-07-10.html`). 사용자가 경로를 지정하면 그 경로.
- 같은 주제 문서가 이미 있으면 새로 짜지 말고 **읽어서 변경분만 갱신**(타임라인 항목 추가 등).
- 다른 산출물(코드·설정)은 건드리지 않는다. HTML 문서 1개만 생성/갱신한다.

### 5. 자기검증
- 단일 HTML로 브라우저에서 열리는가(외부 의존 없음, 이미지가 있으면 상대경로)
- TOC의 각 링크가 실제 섹션 앵커로 이동하는가
- 표시한 사실(파일·커밋·수치)이 실제와 일치하는가
- 지정/기본 경로에 저장됐고, 다른 파일은 변경되지 않았는가

## 재호출 (후속)
대상 문서가 있으면 통째로 다시 만들지 말고 해당 섹션/타임라인만 갱신한다. "이 항목 추가", "다음 단계 갱신" 같은 요청은 그 부분만 외과적으로 수정한다.
