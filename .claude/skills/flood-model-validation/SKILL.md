# Flood Model Validation — 통합 정합성 & 모델 검증 (QA)

이 스킬은 "각각은 맞는데 연결이 틀린" 결함을 잡는다. 지리공간에서 가장 흔한 런타임 오류는 격자 계약 위반이며, 이는 에러 없이 잘못된 위험지도를 만든다. 따라서 검증의 최우선은 **경계면 교차 비교**다.

## 검증 우선순위
1. **격자 계약 정합성** (최우선) — 어긋난 격자는 이후 모든 결과를 침묵 오염.
2. **모델 성능 검증** — 라벨 대비 실제 예측력.
3. **공간 과적합·라벨 누수** — 성능이 진짜인지.
4. **산출물 품질** — 값 범위·NoData·메타데이터.

## 1. 격자 계약 정합성 (양쪽 동시 읽기)
각 인자·라벨·위험지수 래스터를 **마스터 격자와 함께** 열어 5개 불변식을 검사한다. `../geodata-grid-pipeline/scripts/align_to_master_grid.py`의 `assert_grid_contract(master, factor)`를 사용.

체크리스트(계약 §5와 1:1 대응):
- [ ] CRS == EPSG:5179
- [ ] transform == 마스터 transform (원점·픽셀크기 완전 일치)
- [ ] shape == 마스터 shape
- [ ] 셀 중심 정렬(허용오차 ≤ 1e-6 m), 원점 100m snap
- [ ] 마스크 정합(육지밖 NoData 셀에 값 누출 없음)
- [ ] NoData 규약 준수(결측이 0으로 채워지지 않음) — 태그의 standardization_method 추적

> **왜 양쪽을 동시에 읽나:** 인자 하나만 보면 "정상"이다. 마스터와 나란히 놓고 transform·셀중심을 비교해야 100m 밀림·좌표계 차이가 드러난다. 존재 확인("래스터가 있는가")이 아니라 연결 검증("마스터와 같은 셀 위에 있는가")을 한다.

## 2. 모델 성능 검증 (침수흔적도 + NDMS 라벨 대조)
학습에 쓰지 않은 **홀드아웃/공간 CV 폴드**에서 평가한다(같은 셀로 자기평가 금지).

| 지표 | 의미 | 홍수에서 중요한 이유 |
|------|------|-------------------|
| ROC-AUC | 순위 분리력 | 전반적 판별력. 단, 불균형에서 낙관적 |
| **PR-AUC** | 정밀도-재현율 | 침수 극소수 불균형에 더 정직 |
| POD(재현율) | 실제 침수 중 탐지 | 놓친 위험지 = 미경보 |
| FAR | 예측 침수 중 오탐 | 과잉 경보 비용 |
| CSI(위협점수) | POD·FAR 종합 | 기상·홍수 예측 표준 |

라벨 특성 반영: 침수흔적도·NDMS 모두 관측/신고 기반이라 "미기록 = 미침수"가 아니다 → 미탐(FN)의 일부는 미기록 침수일 수 있으므로, 성능 해석 시 라벨 불완전성을 명시한다. NDMS 신고 편향(도시 과대)도 지역별 성능 편차로 드러나는지 확인한다.

## 3. 공간 과적합·라벨 누수 점검
- **공간 CV 확인**: modeler가 무작위 k-fold를 썼다면 공간 누수로 성능이 과대. 유역/블록 CV였는지 검증.
- **인자 중요도 타당성**: HAND·하천거리·확률강우 등 물리 인자가 상위인지. 격자ID·좌표 같은 누수성 변수가 상위면 경고.
- **지역 일반화**: 학습에 없던 지역에서 성능이 급락하는지(진짜 일반화 vs 지역 암기).

## 4. 산출물 품질
- 값 범위 정상(확률 0~1, 음수 없음), NoData 태그 정확, 메타데이터(source/method/crs) 존재.
- 시나리오 예측(재현주기별)이 단조성 상식을 따르는지(대개 200년 ≥ 100년 ≥ 50년 위험). 역전 셀은 원인 조사.

## 출력
- `_workspace/05_validation_report.md` — 통과/실패/미검증 항목 구분. 위반은 파일:항목 + 수정방법 명시.
- 계약 위반 발견 시: 해당 에이전트에게 **구체적 수정 요청**(어느 파일이 어느 불변식을 어떻게 위반, 재표준화 방법). 경계면 이슈는 생산자·소비자 **양쪽**에 알림.

## 증분 검증 (전체 완성 후가 아니라 즉시)
각 모듈 완성 직후 검증한다: 데이터 엔지니어가 자료를 표준화하면 즉시 계약 검증, 인자 분석가가 래스터를 내면 즉시 정렬 확인. 버그를 초기에 잡아야 하류 전파를 막는다. 전체 완성 후 1회 검증은 누적 버그로 수정비용이 커진다.
