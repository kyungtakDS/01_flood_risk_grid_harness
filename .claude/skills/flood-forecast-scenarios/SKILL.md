# Flood Forecast Scenarios — 시나리오별 위험도 예측

현재 상태 위험지도를 넘어, "이만큼 비가 오면 위험이 어떻게 변하는가"를 예측한다. 기본은 **확률강우 재현주기(50/100/200년) 시나리오**다.

## 원리
`flood-risk-modeler`가 학습한 모델은 인자→침수확률 함수다. 예측은 이 함수를 **강우 인자만 시나리오 값으로 교체**해 다시 평가하는 것이다. 지형·노출·취약성은 (단기적으로) 고정, 확률강우량이 재현주기에 따라 커지면 수문 인자(강우강도·범람잠재)가 변해 위험지수가 재산출된다.

## 재현주기 시나리오 (기본)
1. `hydro-hazard-analyst`가 만든 재현주기별 확률강우 격자(`03_hydro_rain_{50,100,200}yr.tif`)를 입력으로 확보.
2. 각 재현주기에 대해 강우 의존 인자를 갱신한 인자 스택 구성(지형·노출은 그대로).
3. 학습 모델로 전 격자 예측 → `04_forecast_{RP}yr.tif` (재현주기별 위험지수).
4. 시나리오 간 **증분·전이** 산출: 예) 100년→200년에서 새로 위험등급이 오르는 셀(한계 노출 지역) 식별. 방재 우선순위에 직접 쓰인다.

> 재현주기 표준셋은 기본 50/100/200년. 사용자가 30년·500년 등을 요청하면 해당 강우 격자를 hydro에 요청해 추가한다.

## 일관성 원칙
- 모든 시나리오 산출은 **같은 마스터 격자·같은 모델**을 사용한다(격자·모델이 바뀌면 시나리오 비교가 무의미).
- 예측도 modeler와 동일하게 **중권역 단위**로 수행하고, 국가 원점 정렬로 이음매 없이 전국 모자이크한다. 중권역별 모델이면 해당 모델을, 대권역 공유 모델이면 그 모델을 중권역 타일에 적용.
- 강우 외 인자를 임의로 바꾸지 않는다. 바꾼다면(예: 개발로 불투수면 증가) 시나리오 정의에 명시하고 별도 트랙으로 관리.
- 산출 후 `assert_grid_contract`로 각 예측 래스터의 마스터 정합 확인.

## 확장 옵션 (조건부)
기본 재현주기 외 요청이 있을 때만 해당 참조를 로드한다:
- **기후변화 장기(SSP)**: 미래 기후 강수 변화를 재현주기 강우에 반영해 장기 적응계획용 위험도 산출 → [`references/climate-ssp.md`](references/climate-ssp.md).
- **실시간 nowcasting**: 기상청 초단기예보·레이더 강우를 실시간 입력으로 수시간 내 위험 예측(운영·경보) → [`references/nowcasting.md`](references/nowcasting.md).

## 출력
- `_workspace/04_forecast_50yr.tif`, `_forecast_100yr.tif`, `_forecast_200yr.tif` (마스터 정렬 COG)
- `_workspace/04_scenario_delta.tif` — 시나리오 간 위험 증분
- `_workspace/04_scenario_summary.md` — 재현주기별 위험면적·인구노출 요약

## 팀 협업 요점
- `flood-risk-modeler`로부터 학습 모델·인자 정의 수신(모델을 새로 학습하지 않음, 재사용).
- `hydro-hazard-analyst`로부터 재현주기별 확률강우 격자 수신. 없는 재현주기는 요청.
- `geo-cartographer`에게: 시나리오별 위험지도화 대상 목록 전달.

## 이전 산출물이 있을 때 (재호출)
`_workspace/04_forecast_*`가 존재하면: 사용자가 재현주기를 바꾸면(예: 200→500년) 해당 시나리오만 추가/교체. 모델이 재학습됐으면 전 시나리오 재예측.
