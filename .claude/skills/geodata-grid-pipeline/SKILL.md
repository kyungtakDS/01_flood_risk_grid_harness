# Geodata Grid Pipeline — 원시자료 → 표준격자망 표준화

전국 100m 홍수위험도의 **토대**를 만드는 스킬. 이종(異種) 원시자료를 단일 표준격자망에 정합시켜, 이후 모든 인자 분석·모델링이 물리적으로 같은 셀 위에서 이뤄지도록 보장한다.

## 왜 이 단계가 가장 중요한가

홍수위험도 격자화의 본질은 "서로 다른 형태로 제작된 자료를 표준격자망에 맞추어 표준화하는 것"이다. 이 단계가 어긋나면 — 좌표계 불일치, 격자 원점 밀림, 해상도 오차, NoData를 0으로 오처리 — 이후 지형·수문·노출 인자 중첩과 ML 학습이 **조용히** 오염된다. `numpy`는 shape만 같으면 좌표가 어긋나도 계산해 버리므로, 이 오염은 에러 없이 잘못된 위험지도를 만든다. 그래서 표준화를 일급 관심사로 다루고, 격자 계약으로 강제한다.

## 격자 계약을 먼저 읽어라

작업 시작 전 **반드시** [`references/grid-standard.md`](references/grid-standard.md)를 읽는다. 이것이 전 에이전트 공통 계약(CRS·해상도·원점·격자ID·NoData·포맷·정합 불변식)이며, 여기서 벗어난 산출물은 검증 단계에서 반려된다.

## 표준 워크플로우

### Step 1 — 마스터 격자 생성 (가장 먼저)
분석 대상 범위에 대해 `grid_master.tif`를 생성한다. 전국은 **수자원단위지도 대권역(한강·낙동강·금강·영산강·섬진강)** 단위로 나눠 처리한다(`grid_master_{daegwon}.tif`).
- CRS=EPSG:5179, 100m, 원점은 **국가 표준 격자에 snap**된 좌표(임의 bbox 좌하단 금지 — 유역 경계에서 반셀씩 어긋나 전국 모자이크가 불가능해진다).
- 육지/도서 = 유효셀, 해양·국외 = NoData. 이 마스크가 곧 분석 대상.
- 격자 속성에 유역코드(`daegwon`·`junggwon`·`std_watershed`)를 부착한다. 중권역은 ML 모델링·공간 CV 단위가 된다.
- 이후 모든 래스터는 이 파일의 CRS·transform·shape를 **복사**해 정렬한다. "격자를 새로 만들지 말고 마스터에 맞춰라."
- 유역 위계·파티셔닝·완전유역 처리 규칙은 [`references/watershed-units.md`](references/watershed-units.md) 참조.

### Step 2 — 입력 형태 판별 (원시 vs 이미 격자화)
각 입력 자료가 **① 원시자료(벡터/래스터/집계)**인지 **② 이미 100m 격자로 제작된 자료**인지 먼저 구분한다. 같은 자료(예: 침수위험지도 폴리곤, NDMS 피해 포인트)라도 두 형태로 들어올 수 있다.

- **원시자료** → 기하 유형과 원본 CRS를 확인(벡터는 CRS 태그가 없거나 틀린 경우가 흔하므로 메타데이터·육안 교차 확인, 무단 가정 금지) → Step 3 유형별 표준화.
- **이미 격자화된 자료** → 그대로 신뢰하지 말고 `assert_grid_contract(master, input)`로 **격자 계약 준수를 검증**한다. 남이 만든 100m 격자는 원점·CRS 변형(EPSG:5174/5186 등)·NoData 규약·정렬이 마스터와 다를 수 있다.
  - 계약 통과 → 재표준화 없이 그대로 사용(불필요한 리샘플로 값을 훼손하지 않는다).
  - 계약 위반(원점 밀림·CRS 상이 등) → 마스터에 재정합(연속형/범주형 리샘플 규칙 준수). 재정합 사실을 태그에 기록.

> **왜 검증하나:** "이미 100m 격자니까 맞겠지"가 가장 흔한 함정이다. 반셀 밀린 격자, 미묘하게 다른 CRS, 다른 NoData 값은 육안으로 안 보이지만 셀 단위 join을 조용히 어긋나게 한다. 계약 검증은 값싸고, 오염 전파는 비싸다.

### Step 3 — 유형별 표준화 방법 적용

이 표가 스킬의 심장부다. 자료 유형과 **의미**에 맞는 방법을 선택한다.

| 원시자료 유형 | 대표 자료 | 표준화 방법 | 왜 / 함정 |
|--------------|----------|------------|----------|
| 벡터 폴리곤 | 토지피복, 침수흔적도, 행정경계 | 면적가중 집계 / 우세값(majority) / 피복률(%) | 셀에 여러 폴리곤이 걸치면 우세값 또는 비율로. 단순 중심점 판정은 경계 셀 왜곡 |
| 벡터 포인트 | 강우관측소, 건물 | 개수·밀도 집계 / 공간보간(IDW·Kriging) | 강우처럼 연속장은 보간, 시설처럼 개별객체는 밀도. 혼동 금지 |
| 벡터 라인 | 하천망, 제방, 도로 | 거리변환(distance-to) / 라인밀도 | 하천 근접위험은 거리변환. 셀에 라인이 안 닿아도 거리는 전 셀에 정의됨 |
| 연속 래스터(상이 해상도) | DEM, 레이더 강수 | 재투영 → 마스터 정합 → **bilinear/cubic** | 고→저해상도 다운샘플 시 목적 반영(저지대 강조는 min pooling 고려) |
| 범주 래스터 | 토지피복 코드, 토양그룹 | 재투영 → **nearest/majority** | **bilinear 절대 금지** — 존재하지 않는 코드가 보간으로 생성됨 |
| 통계·집계(행정단위) | 인구·자산·취약계층 | **다이어시메트릭 분배**(건물/불투수면 가중 하향식) | 행정단위 균등분배는 비거주지에 인구를 뿌림. 가중 분배로 질량 보존 |
| 시계열 | 강우·수위 시계열 | 시간집계 → 빈도분석 → 격자 매핑 | 확률강우량은 빈도분석 후 격자화. 원시 시계열을 바로 격자화하지 않음 |

> 반복·결정적 작업(재투영·snap·정합·검증)은 [`scripts/align_to_master_grid.py`](scripts/align_to_master_grid.py)에 번들링되어 있다. `align_raster`, `align_vector`, `dasymetric_disaggregate`, `assert_grid_contract`를 임포트해 사용하거나 확장한다.

### Step 4 — 침수이력 라벨 격자화 (ML 타깃)
ML 모델(RandomForest·XGBoost·LightGBM·CatBoost·Decision Tree)의 학습 타깃을 만든다. 라벨 자료(침수흔적도·침수위험지도 폴리곤, NDMS 피해 포인트)는 **원시 형태로 올 수도, 이미 100m 격자로 제작되어 올 수도** 있다. Step 2의 형태 판별을 그대로 적용한다:
- **이미 격자화된 라벨** → `assert_grid_contract`로 검증 후 그대로 사용(통과 시), 위반 시 재정합.
- **원시 라벨** → 아래 방법으로 격자화. **두 출처를 결합**한다:
  - **침수흔적도/침수위험지도(폴리곤)** → 격자 침수여부(1). 면적/중심 판정으로 셀에 매핑.
  - **NDMS(국가재난관리시스템) 피해자료(포인트)** → 피해지점 좌표를 셀에 매핑(1). 지오코딩 오차는 소반경 버퍼로 완화. 피해 건수·심도는 가중 라벨/보조 회귀 타깃으로도 보존.
- 두 출처의 **합집합**을 양성(1)으로 하고, 각 셀에 출처(폴리곤/포인트)·연도·사상을 태그로 남긴다(검증자가 라벨 신뢰도 추적).
- 라벨도 **마스터 격자에 정렬**되어야 인자와 join된다.
- 침수(1)는 극소수 → 심각한 불균형. 비침수(0) 표본 설계는 modeler와 협의(지형만 외우지 않도록 "침수가능 지형의 실제 미침수"를 우선 샘플링, NDMS 신고 편향 보정). 자료 출처는 [`references/kr-data-sources.md`](references/kr-data-sources.md) §6 참조.

### Step 5 — 자기 검증 (증분 QA 전 셀프체크)
각 표준화 산출물에 대해 `assert_grid_contract(master, factor)`를 호출해 5개 불변식(CRS·transform·shape·셀정렬·마스크정합)을 스스로 확인한 뒤 다음 자료로 넘어간다. 위반을 안고 진행하면 오염이 하류로 전파된다.

## NoData 원칙 (반복 강조할 가치가 있음)
**결측 ≠ 0.** 강우 결측을 0mm로 채우면 "비 안 왔음"이 되어 위험도가 왜곡된다. 결측은 결측으로 전파하고, 대치/제외는 모델 단계에서 명시적으로 결정한다. 각 인자의 NoData 처리 방식을 래스터 태그에 기록해 검증자가 추적하게 한다.

## 출력
- `_workspace/02_grid/grid_master.tif` — 마스터 격자(마스크)
- `_workspace/02_grid/{source}.tif` — 표준화된 각 원시자료 (단일밴드 COG, EPSG:5179, 태그에 표준화방법 기록)
- `_workspace/02_grid/labels.tif` — 침수이력 라벨(1/0)
- 각 파일 메타: `source`, `standardization_method`, `nodata`, `crs=EPSG:5179`

## 전국 자료원
자료별 출처·형식·주의는 [`references/kr-data-sources.md`](references/kr-data-sources.md) 참조 (지형·하천/유역·강수·토지피복/토양·인구/자산·침수이력).

## 이전 산출물이 있을 때 (재호출)
`_workspace/02_grid/`가 이미 존재하면: 새 자료만 추가 표준화하고, 사용자 피드백이 지정한 자료만 재처리한다. 마스터 격자는 재생성하지 말고 재사용한다(재생성 시 하류 인자와 정렬이 깨진다). 마스터를 반드시 바꿔야 하면 전 인자 재정합이 필요함을 리더에게 알린다.
