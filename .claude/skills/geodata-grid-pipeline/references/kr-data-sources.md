# 전국 홍수위험 원시자료 소스 (대한민국)

홍수위험도 격자화에 쓰이는 전국 단위 원시자료의 출처·형식·표준화 시 주의점. 자료명·기관은 변경될 수 있으므로, 실제 수집 시 최신 포털을 확인한다. 이 문서는 "어떤 자료를 어디서, 어떤 형식으로 받아, 어떤 표준화가 필요한지"의 지도다.

## 목차
1. 지형·수치표고
2. 하천·유역·수문
3. 강수·기상
4. 토지피복·토양
5. 인구·자산·행정
6. 홍수이력·침수(ML 라벨)

---

## 1. 지형·수치표고 (terrain-hazard-analyst 사용)

| 자료 | 기관/포털 | 형식 | 표준화 주의 |
|------|----------|------|-------------|
| 수치표고모델(DEM) 5m/30m | 국토지리정보원(NGII) 국토정보플랫폼 | 래스터(GeoTIFF/IMG) | 연속형 → bilinear/cubic 리샘플. 5m를 100m로 낮출 때 평균/최솟값 중 목적에 맞게 선택(저지대 강조는 min pooling 고려) |
| 수치지형도(등고선·표고점) | NGII | 벡터 | DEM 보간의 원천. 이미 DEM이 있으면 생략 |

## 2. 하천·유역·수문 (hydro-hazard-analyst 사용)

| 자료 | 기관/포털 | 형식 | 표준화 주의 |
|------|----------|------|-------------|
| 하천망(국가·지방하천) | WAMIS(수자원관리종합정보), 하천관리지리정보(RIMGIS) | 벡터 라인 | 거리변환(distance-to-river)·차수(Strahler). 라인 → 격자 거리 래스터 |
| 표준유역 경계 | WAMIS | 벡터 폴리곤 | 수문 인자 집계 단위. terrain의 흐름누적과 정합 필요 |
| 하천 수위·유량 관측 | WAMIS, 홍수통제소 | 시계열/포인트 | 빈도분석·보간 |
| 제방·수리시설 | RIMGIS, 한국농어촌공사 | 벡터 | 방어시설 → 취약성 완화 인자 |

## 3. 강수·기상 (hydro-hazard-analyst 사용)

| 자료 | 기관/포털 | 형식 | 표준화 주의 |
|------|----------|------|-------------|
| ASOS/AWS 강수 관측 | 기상청 기상자료개방포털(KMA) | 포인트 시계열 | 지점 → 격자 보간(IDW/Kriging/PRISM). 지형보정 고려 |
| 레이더 강수(HSR/합성) | 기상청 | 래스터(상이 해상도) | 재투영 → 마스터격자 정합 |
| 확률강우량(빈도별) | 국가 확률강우량도, 홍수통제소 | 격자/표 | 재현주기(50/100/200년)별. 이미 격자면 정합만, 지점값이면 보간 |
| 초단기·단기예보 | KMA(동네예보 API) | 격자 | nowcasting 확장 시 사용 |

## 4. 토지피복·토양 (exposure/hydro 공용)

| 자료 | 기관/포털 | 형식 | 표준화 주의 |
|------|----------|------|-------------|
| 토지피복지도(대/중/세분류) | 환경부 환경공간정보서비스(EGIS) | 벡터 폴리곤/래스터 | **범주형** → nearest/majority. bilinear 금지. 불투수면 비율 산출에 사용 |
| 정밀토양도 | 농촌진흥청 흙토람 | 벡터 폴리곤 | 유출곡선지수(CN) 산정용. 수문학적 토양그룹(A~D) |

## 5. 인구·자산·행정 (exposure-vulnerability-analyst 사용)

| 자료 | 기관/포털 | 형식 | 표준화 주의 |
|------|----------|------|-------------|
| 100m 격자 인구·가구 통계 | 통계청 SGIS(통계지리정보) | 격자/집계구 | **이미 표준격자와 동일 체계** → 정합만. 가장 직접적인 노출 자료 |
| 행정구역 단위 인구·자산 | 통계청 KOSIS | 표(행정단위) | **다이어시메트릭 분배** — 건물/불투수면 가중 하향식으로 격자화 |
| 건물통합정보 | 국토부 브이월드(V-World), 건축물대장 | 벡터 폴리곤/포인트 | 밀도·연면적 집계. 다이어시메트릭 가중치로도 사용 |
| 사회취약계층(고령·독거 등) | KOSIS, 행정안전부 | 표(행정단위) | 취약성 지표. 다이어시메트릭 분배 |

## 6. 홍수이력·침수 — ML 라벨 (geo-data-engineer가 격자화, modeler가 학습)

ML(RF/XGBoost)의 학습 라벨. **라벨 품질이 모델 성패를 좌우**하므로 최우선 확보 대상.

| 자료 | 기관/포털 | 형식 | 표준화 주의 |
|------|----------|------|-------------|
| 침수흔적도 | 행정안전부, 각 지자체 | 벡터 폴리곤 | 폴리곤 → 격자 침수여부(1). 연도·사상별 구분. **주 타깃 라벨** |
| NDMS 피해자료 | 국가재난관리시스템(행정안전부) | 벡터/표 **포인트** | 피해지점 좌표 → 격자 매핑(침수/피해=1). **주 타깃 라벨**. 지오코딩 오차는 소반경 버퍼로 완화, 피해 건수·심도는 가중/회귀 타깃 활용. 신고 편향(도시 과대) 유의 |
| 홍수위험지도 | 홍수위험지도정보시스템(FIMS, 환경부) | 벡터/래스터 | 재현주기별 침수예상범위. 라벨 보강·검증 양쪽 활용 |
| 재해연보·피해이력 | 행정안전부 | 표/문서 | 위치 지오코딩 후 격자 매핑 |
| 위성/SAR 침수관측 | KARI, 국내외 위성 | 래스터 | 실측 침수 라벨 보강. 재투영·정합 |

**주 타깃(라벨) 결합:** ML 학습의 양성(1) 클래스는 **침수흔적도/침수위험지도(폴리곤) ∪ NDMS 피해자료(포인트)**의 합집합으로 구성한다. 폴리곤은 침수 범위를, 포인트는 개별 피해 발생을 준다. 두 출처는 상호 보완적이며(폴리곤 미작성 지역을 포인트가 보강), 각 셀에 출처·연도·사상을 태그로 남겨 신뢰도를 추적한다.

**제공 형태(원시 vs 격자):** 이 라벨 자료들은 **원시자료(폴리곤/포인트)로 제공될 수도, 이미 표준격자망에 맞춰 제작된 100m 격자로 제공될 수도** 있다. 격자로 제공되면 `assert_grid_contract`로 마스터 정합을 검증한 뒤(통과 시) 그대로 사용하고, 원시로 제공되면 격자화한다. 격자 제공본이라도 원점·CRS·NoData가 마스터와 다를 수 있으므로 무검증 사용은 금지.

**라벨 균형:** 침수(1) 셀은 전국 격자의 극소수 → 심각한 클래스 불균형. 비침수(0) 표본은 무작위가 아니라 "침수 가능 지형(하천 인근·저지대)에서 실제 미침수"를 우선 샘플링해야 모델이 지형만 외우지 않는다. 자세한 표본 설계는 `flood-risk-index-modeling` 스킬 참조.

## 표준화 시 공통 확인
- 모든 자료의 원본 CRS를 먼저 확인(각기 다름: WGS84, UTM-52N, EPSG:5174/5186 등) → **EPSG:5179로 재투영**.
- 벡터는 좌표계 태그가 없거나 잘못 지정된 경우가 흔하다 → 반드시 육안·메타데이터로 교차 확인.
- 다운로드 시점의 자료 버전·연도를 메타데이터에 기록(감사 추적).
