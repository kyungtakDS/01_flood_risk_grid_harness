# Flood Risk Index Modeling — 인자 통합 위험지수 (ML)

지형·수문·노출·취약성 인자를 결합해 격자별 홍수위험지수(=침수 확률/위험등급)를 산출한다. 기본 방법은 **여러 트리 기반 머신러닝 모델을 학습·비교**하여 과거 침수이력을 학습하는 데이터 주도 접근이다.

## 모델군 (학습·비교 후 선택/앙상블)
아래 5개 모델을 동일한 학습표·동일한 공간 CV로 학습하고 성능을 비교한다. 트리 계열이라 인자 스케일에 강건하고 비선형·상호작용을 포착한다.

| 모델 | 특성 | 홍수 모델링에서의 강점 |
|------|------|----------------------|
| Decision Tree | 단일 트리, 해석 용이 | 베이스라인·규칙 해석. 단독 성능은 낮음(과적합) |
| RandomForest | 배깅 앙상블 | 안정적, 과적합에 강건, permutation 중요도 |
| XGBoost | 부스팅 | 높은 정확도, 정규화·결측 자체 처리 |
| LightGBM | 리프중심 부스팅 | 대용량 전국 격자에 빠르고 메모리 효율적 |
| CatBoost | 순서형 부스팅 | 범주형(토양그룹·토지피복 코드) 자체 처리, 과적합에 강건 |

**비교·선택 원칙:** 단일 지표(AUC)만 보지 말고 공간 CV의 지역별 편차, PR-AUC(불균형 반영), 인자 중요도의 물리적 타당성을 함께 본다. 최종은 최고 모델 단독 또는 상위 모델 소프트보팅 앙상블로 결정하고, 근거를 모델 카드에 기록한다. 범주형 인자를 원핫 없이 쓰려면 CatBoost/LightGBM의 native categorical을 활용한다.

## 입력
- 인자 래스터 스택: `_workspace/03_terrain_*`, `03_hydro_*`, `03_expo_*` (모두 마스터 격자 정렬).
- **타깃(라벨)**: `_workspace/02_grid/labels.tif` (침수=1 / 비침수=0). data-engineer가 준비하며, 원시(폴리곤/포인트) 또는 이미 격자화된 형태 어느 쪽이든 마스터 격자에 정합된 상태로 받는다. 두 출처를 결합:
  - **침수흔적도/침수위험지도(폴리곤)** → 셀 침수여부(면적·중심 판정).
  - **NDMS(국가재난관리시스템) 피해자료(point)** → 피해지점 좌표를 격자에 매핑해 침수/피해 셀(1). 지오코딩 오차가 있는 포인트는 소반경 버퍼로 완화하고, 피해 건수·심도는 가중 라벨/보조 회귀 타깃으로도 활용 가능.
  - 두 출처의 **합집합**을 양성(1)으로 하되, 출처(폴리곤/포인트)·연도·사상 정보를 태그로 보존해 검증자가 라벨 신뢰도를 추적하게 한다.
- **전제**: 모든 인자·라벨이 동일 마스터 격자에 정렬되어 있어야 셀 단위 join이 성립. 시작 전 `assert_grid_contract`로 스택 정합을 확인한다(어긋난 인자가 하나라도 있으면 학습표가 지리적으로 뒤섞인다).

## 학습표 구성
1. 인자 스택을 격자 ID 기준으로 넓은 테이블(GeoParquet)로 합류: 행=셀, 열=인자 + 라벨.
2. NoData 셀 처리: 결측을 명시적으로 대치(중앙값/모델 기반)하거나 제외. **결측을 0으로 채우지 않는다**(강우 0 = 오해).
3. 정규화: 트리 모델(RF/XGBoost)은 스케일 불변이라 필수는 아니나, 이상치 클리핑은 유지.

## 클래스 불균형 — 라벨 표본 설계 (성패의 핵심)
침수(1) 셀은 전국의 극소수라 심각한 불균형이다.
- **음성 표본(0) 설계가 중요**: 전 격자 무작위 추출은 대부분 산·고지대라 모델이 "고도만 높으면 안전"을 외운다. 대신 **침수 가능 지형(하천 인근·저지대·저 HAND)에서 실제 미침수**를 우선 표본으로 뽑아 어려운 음성을 학습시킨다.
- 불균형 대응: class_weight/scale_pos_weight, 언더샘플링, 또는 focal 계열. 임계값은 F1/기대비용 기준으로 조정.
- 라벨 신뢰도: 침수흔적도·NDMS 피해자료 모두 **관측·신고된 사상에 한정**되므로 "미기록 = 미침수"가 아님을 유의(검증자와 공유). NDMS 포인트는 신고 편향(도시·인구밀집지에 과대)이 있을 수 있어, 이를 음성 표본 설계 시 보정한다.

## 중권역 단위 모델링 (부하 저감 + 공간 일관성)
ML 모델링은 수자원단위지도 **중권역** 단위로 수행한다. 전국 수백만 셀을 한 번에 학습하지 않고 중권역 블록으로 나눠 메모리·연산 부하를 줄이며, 중권역은 수문학적으로 응집돼 인자-침수 관계가 지역 내 일관된다.
- 라벨 밀도에 따라 전략 선택: 중권역 라벨이 충분하면 **중권역별 개별 모델**, 희소하면 **대권역 공유 모델 + 중권역 예측**으로 상향 조정. 상세는 `../geodata-grid-pipeline/references/watershed-units.md` §4 참조.
- 중권역별 예측을 국가 원점 정렬 덕에 이음매 없이 모자이크 → 전국 위험지수. 유역 경계 셀 이중계산 금지(검증자 확인).

## 공간 교차검증 (반드시)
일반 무작위 k-fold는 인접 셀이 train/test에 섞여 **공간 누수**로 성능이 과대평가된다. **중권역을 자연 fold로 하는 spatial block CV**를 사용한다(중권역 = 처리 단위이자 CV 블록). 이렇게 해야 "학습에 없던 유역"에 대한 일반화를 정직하게 평가한다.

## 모델링 절차
1. RF 또는 XGBoost 기본 모델 학습 → 공간 CV로 성능(AUC/PR-AUC) 확인.
2. 하이퍼파라미터 튜닝(트리수·깊이·학습률·정규화). 과적합 징후는 검증자와 교차 확인.
3. 전 격자 예측 → 침수 확률 래스터(0~1) = 홍수위험지수.
4. **인자 중요도**(gain/permutation/SHAP) 산출 → 물리적으로 타당한지 점검(예: HAND·하천거리·확률강우가 상위여야 자연스러움). 비물리적 인자가 지배하면 라벨 누수·표본 편향을 의심.
5. **불확실성**: 트리 분산/부트스트랩/예측구간으로 셀별 신뢰도 래스터 산출.

## 보조: 다기준 가중지수(AHP/엔트로피)
라벨이 부족한 지역이나 해석 가능성이 필요할 때, 인자를 정규화·가중합하는 결정론적 지수를 병행 산출할 수 있다. ML 결과와 상호 비교하면 신뢰도가 높아진다(둘이 크게 다른 지역은 재검토 대상). 가중치는 AHP(전문가 쌍대비교) 또는 엔트로피(데이터 기반). 이 방식은 옵션이며 기본은 ML이다.
